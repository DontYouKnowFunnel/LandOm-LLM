## DEFINITIONS (STRICT INTERPRETATION)

- HERO:
  The top-most section, visible without scrolling.
  Includes a main headline, supporting text, and a primary CTA, often with a strong visual.

- VALUE_PROP:
  Explains the core value or benefits of the product ("why it matters").
  Uses persuasive language rather than listing features.

- FEATURE:
  Describes specific product features or capabilities ("what it does").
  Often shown as a list, grid, or repeated blocks.

- SOCIAL_PROOF:
  Shows that other users trust or use the product.
  Includes testimonials, reviews, ratings, or user counts.

- TRUST:
  Builds credibility through authority signals.
  Includes certifications, security badges, compliance, or guarantees (not user opinions).

- PRICING:
  Displays pricing plans, tiers, or cost-related details.
  Often includes numbers, currency, and comparison between plans.

- CTA_SECTION:
  A dedicated section designed to drive user action.
  Typically includes strong CTA text with minimal additional content.

- INPUT_FORM:
  A section that collects user input such as email, signup info, or survey responses.
  Identified by input fields, dropdowns, or text areas.

- CHECKOUT:
  A transactional section where users complete a purchase.
  Includes billing details, payment inputs, and order summary.

- INTERACTIVE:
  Sections with UI elements that respond to user interaction (e.g., tabs, sliders, calculators).
  Often allows users to preview, simulate, or experience the product before conversion.

- FAQ:
  A structured list of questions and answers.
  Often uses expandable or collapsible items.

- POPUP:
  Overlay UI elements (modals, dialogs, exit-intent popups) that appear above the page.
  Interrupt the normal flow to capture attention or drive immediate action.
