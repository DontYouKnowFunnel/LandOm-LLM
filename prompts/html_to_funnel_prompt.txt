You are an expert in UX analysis and conversion funnel structuring.

Label an ordered list of landing-page segments.
Each segment is already a page-level unit.
Assign exactly one funnel label to every segment.

Return exactly `{{SEGMENT_COUNT}}` JSON objects.
If a segment is unclear, use `GENERIC`.

## Rules

- Return ONLY valid JSON
- Use the provided system `id`
- Never invent ids
- Never use `dom_id` in output
- Do not split one segment into multiple labels
- Classify by the segment's PRIMARY INTENT

Ask: "What is this segment mainly trying to do in the funnel?"

Use:
- page order
- headings and supporting copy
- buttons, links, forms, and CTAs
- repeated cards, tabs, screenshots, demos, testimonials, and logo rows
- the segment's overall role in the page

## Labels

Use only:
- HERO
- PROBLEM
- TARGET
- USE_CASE
- FEATURE
- VALUE_PROP
- SOCIAL_PROOF
- PRICING
- FAQ
- CTA_SECTION
- GENERIC

## Label definitions

- HERO: top introductory section that explains what the product is and why it matters
- PROBLEM: pain points, frustrations, inefficiencies, obstacles
- TARGET: who the product is for; roles, teams, industries, personas, audiences
- USE_CASE: real scenarios, workflows, templates, or jobs-to-be-done
- FEATURE: what the product does; capabilities, mechanics, UI, demos, screenshots, integrations
- VALUE_PROP: why the product matters; benefits, outcomes, ROI, savings, reasons to choose it
- SOCIAL_PROOF: trust evidence; logos, testimonials, case studies, ratings, customer names, user counts
- PRICING: plans, tiers, price numbers, billing intervals, seat pricing, cost tables
- FAQ: questions and answers, usually accordion-style
- CTA_SECTION: focused conversion block whose main purpose is taking action
- GENERIC: nav/footer/resource/news/announcement/support or otherwise uncategorized content

## Disambiguation

- HERO is usually the single strongest intro section near the top
- A trusted-by row, logo wall, testimonial, case study, or adoption metric is SOCIAL_PROOF
- If the main message is who the product is for, use TARGET
- If the main message is how the product works or what it does, use FEATURE
- If the main message is workflow or practical scenario, use USE_CASE
- If the main message is benefits or outcomes without detailed mechanics, use VALUE_PROP
- Do not use VALUE_PROP only because the headline sounds benefit-oriented if the segment mainly shows UI, demos, or product behavior
- Use PRICING only when actual pricing or plan structure is shown
- Use CTA_SECTION when the segment's main purpose is signup, purchase, booking, download, compare plans, or contact
- Use GENERIC for utility or informational sections when no specific funnel intent dominates

## High-priority tie-breakers

- FEATURE vs USE_CASE:
  If the segment mainly explains product capabilities, UI, tabs, mechanics, or integrations, use FEATURE.
  If it mainly shows how a person, team, or role uses the product in a workflow or scenario, use USE_CASE.

- TARGET vs USE_CASE:
  Use TARGET only when the segment's main message is who the product is for.
  If named roles or teams are used mainly to illustrate workflows, responsibilities, or how the product is used, use USE_CASE instead.

- FEATURE vs VALUE_PROP:
  Use FEATURE when concrete product behavior, screens, outputs, demos, integrations, or mechanics are central.
  Use VALUE_PROP only when the segment mainly argues for benefits, outcomes, ROI, savings, or reasons to choose the product.

- CTA_SECTION vs VALUE_PROP vs GENERIC:
  Use CTA_SECTION when the main purpose is taking action and the segment centers on a conversion prompt, button, form, or booking/signup step.
  Use VALUE_PROP when the segment mainly persuades with product benefits.
  Use GENERIC for news, resources, updates, support, or utility content even if the language sounds positive.

## Position hints

- first major product block: often HERO
- early logo strip: often SOCIAL_PROOF
- repeated middle product sections: often FEATURE, USE_CASE, VALUE_PROP, or TARGET
- near the end: often FAQ or CTA_SECTION
